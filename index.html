Session notes

세션별 읽기 노트

Session 0

성공적인 취업 준비
  • 대학일자리플러스센터는 청년 특화 고용 서비스 사업이다.
  • 고용24에서 대학일자리플러스센터를 검색해 찾을 수 있다.
  • 백석대학교 BEST를 통해 비대면 서비스도 확인할 수 있다.
  • 챙길 정책: 국민취업지원제도, 미래내일 일경험, 국민내일배움카드.
Session 1

AI가 다 해주네, 그럼 난 뭐해
  • 개발자는 만드는 사람에서 맡기고 검토하는 사람으로 역할이 바뀌고 있다.
  • AI에게 맡기면 안 되는 일: 채용 최종 결정, 성과 피드백, 아키텍처, 보안 정책, 제품 방향.
  • AI는 추상화다. 장애 상황에서는 구조, 라이브러리 선택 이유, 기본기를 알아야 복구할 수 있다.
  • 결론은 위임 판단과 기본기다. 맡길 것은 맡기되 남는 시간에 위임할 수 없는 판단을 해야 한다.

질문 메모: AI로 만든 코드를 인수인계할 때 이해도를 어떻게 보장할지, 레거시 정보가 사라지는 상황에서 어떻게 수정할지.

Session 2

DEEPDIVE: 하네스엔지니어링
  • 코드는 test, lint, build, QA까지 확인되어야 한다.
  • AI와 개발할 때 가장 중요한 것은 모호성을 줄이는 일이다.
  • 여러 목표를 이어갈 때는 `ultragoal`, 이슈를 작게 쪼갤 때는 Ralph를 쓰는 흐름이 언급됐다.
Session 3

요약 에이전트 솔론봇, 2년 동안 운영한 썰
  • 가설을 세우면 실패도 데이터가 된다.
  • 사이드 프로젝트를 만들기 전 "누구에게", "어디에서" 보일지 먼저 정해야 한다.
  • 흥미성 기능은 사용률이 빠르게 떨어지고, 정보성 기능이 더 오래 간다.
  • 실패한 기능은 홍보 부족보다 트리거, 사용자 맥락, 도배 위험 때문에 실패할 수 있다.
Session 4

러닝 머신이 아니라 머신 러닝이 다이어트에 미치는 영향
  • 브로카 공식처럼 단순한 경험식에서 BMI와 머신러닝 기초 설명으로 이어졌다.
  • 언어모델 결과는 토큰 샘플링 방식에 따라 달라진다.
  • 현재의 인공 신경망은 인간 뇌를 그대로 복제한 구조라기보다 수학적 방식으로 문제를 푸는 구조에 가깝다.
Session 5

RAG는 처음이라: 백엔드 개발자의 LLM 적응기
  • 문의 게시판 답변에서 답변 작성보다 근거 탐색에 더 많은 시간이 든다는 문제가 출발점이었다.
  • 기본 흐름은 문서, 청킹, 임베딩, Vector DB, 검색, 컨텍스트, LLM 답변이다.
  • 중복 문서를 묶고 최신 또는 최장 데이터를 남기는 방식으로 참고 자료를 개선했다.
  • 문서 타입별 검색 할당량을 두어 대화형 데이터와 공식 문서 데이터의 다양성을 확보했다.
  • 6가지 기준으로 자동 채점하는 평가 시스템과 골든 데이터셋을 구축했다.
  • 핵심 결론: 모델보다 데이터 파이프라인과 평가 체계가 중요하다.

개인 질문 후보: 내 LLM Wiki에는 문서 타입별 할당량을 어떻게 적용할 수 있을까?

Session 6

AI 시대를 여행하는 PM을 위한 안내서
  • 좋은 모델 자체가 목표는 아니다. 단 한 명의 end user 요구 하나를 잡아야 한다.
  • 테스트 항목을 읽으면 서비스 요구사항이 전부 파악되어야 한다.
  • AI가 필요한 지식을 얻도록 목표에 맞는 지식과 도구를 넣어줘야 한다.
  • AI 서비스는 rubric, trajectory, observability를 확보해야 한다.
  • 프로젝트는 목표를 잊지 않고 적당한 선에서 멈추는 감각이 중요하다.
Follow-up

내 후속 정리 포인트

위임하지 않을 판단 목록

AI로 작업할 때도 제품 방향, 보안 정책, 아키텍처 선택, 최종 품질 판단은 직접 기준을 잡아야 한다.

RAG 개선 실험

내 지식검색에는 중복 문서 정리, 최신 문서 우선, 문서 타입별 할당량, 골든 질문셋을 작은 실험으로 붙일 수 있다.

프로젝트 목표 축소

각 포트폴리오 프로젝트를 단 한 명의 사용자와 요구 하나로 다시 써보면 범위가 선명해진다.

실패 데이터화

기능을 만들기 전 "누가", "어디서", "어떤 트리거로" 쓸지 적어두면 실패 후 원인을 분석할 수 있다.
