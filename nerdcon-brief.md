## 세션별 메모 요약

### 세션 0. 성공적인 취업 준비

- 대학일자리플러스센터는 청년 특화 고용 서비스 사업이다.
- 고용24에서 대학일자리플러스센터를 검색해 찾을 수 있다.
- 백석대학교 BEST를 통해 비대면 서비스도 확인할 수 있다.
- 청년 정책 키워드: 국민취업지원제도, 미래내일 일경험, 국민내일배움카드.

### 세션 1. AI가 다 해주네, 그럼 난 뭐해

- 개발자는 만드는 사람에서 맡기고 검토하는 사람으로 역할이 이동 중이다.
- AI에게 맡기면 안 되는 일: 채용 최종 결정, 성과 피드백, 아키텍처, 보안 정책, 제품 방향.
- AI는 추상화다. 평소에는 몰라도 쓰지만 장애 상황에서는 내부 구조와 기본기를 알아야 복구할 수 있다.
- 결론: 위임할 대상을 판단하는 능력과 기본기가 여전히 중요하다.

### 세션 2. DEEPDIVE: 하네스엔지니어링

- AI와 개발할 때 중요한 것은 모호성을 줄이는 일이다.
- 코드 품질 확인은 test, lint, build, QA를 포함해야 한다.
- 여러 목표를 이어갈 때는 `ultragoal`, 이슈를 쪼갤 때는 Ralph 실행이 더 적합하다는 맥락이 있었다.

### 세션 3. 요약 에이전트 솔론봇

- 가설을 세우면 실패도 데이터가 된다.
- 사이드 프로젝트는 "누구를 대상으로 하는지", "어디에서 보일지"를 먼저 정해야 한다.
- 흥미성 기능은 사용률이 빠르게 줄고, 정보성 기능이 더 오래 간다.
- 실패한 기능은 홍보 부족이 아니라 트리거, 사용자 맥락, 도배 위험 같은 사용 조건이 문제일 수 있다.

### 세션 4. 머신 러닝과 다이어트

- 브로카 공식처럼 단순한 경험식에서 BMI와 머신러닝 기초로 이어지는 설명.
- 언어모델 결과는 토큰 샘플링 방식에 따라 달라진다.
- 현재의 인공 신경망은 인간 뇌의 직접 복제라기보다 수학적 구조로 문제를 푸는 방식에 가깝다.

### 세션 5. RAG는 처음이라

- 문의 게시판 답변에서 답변 작성보다 근거 탐색이 더 오래 걸리는 문제가 출발점이었다.
- 기본 흐름: 문서, 청킹, 임베딩, Vector DB, 검색, 컨텍스트, LLM 답변.
- 개선 포인트: 중복 문서 정리, 최신/최장 데이터 우선, 문서 타입별 검색 할당량.
- RAG 품질은 모델보다 데이터 파이프라인과 평가 체계가 좌우한다.

### 세션 6. AI 시대를 여행하는 PM을 위한 안내서

- 좋은 모델 자체가 목표가 아니다. 단 한 명의 end user 요구 하나를 분명히 잡아야 한다.
- 테스트 항목을 읽으면 서비스 요구사항이 파악되어야 한다.
- AI 서비스에서는 rubric, trajectory, observability가 중요하다.
- 프로젝트는 목표를 잊지 않고 적당한 선에서 멈추는 감각이 필요하다.
